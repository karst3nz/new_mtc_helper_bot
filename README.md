## ✨ Возможности

### Основные функции
- 📅 **Расписание занятий** - просмотр расписания для 40+ групп МГТЭК с автоматическим парсингом с сайта bseumtc.by
- ⏰ **Учет пропущенных часов** - отслеживание посещаемости с историей изменений
- 👥 **Многогрупповость** - поддержка основной и дополнительной группы для студентов
- 🔔 **Система уведомлений** - ежедневное расписание, напоминания о парах, уведомления о пропусках
- 📊 **Аналитика** - графики посещаемости, статистика за 30 дней, недельное сравнение
- 📥 **Экспорт данных** - выгрузка истории пропусков и изменений расписания в Excel
- ⚙️ **Гибкие настройки** - персонализация для пользователей и групповых чатов
- 🎨 **Современный интерфейс** - эмодзи для предметов, inline-календарь, избранное
- 🚀 **Быстрый доступ** - inline-режим
- 🔧 **Административная панель** - управление ботом, рассылки, статистика

### Технические особенности
- **Асинхронная архитектура** - на базе aiogram 3.x и asyncio
- **База данных** - SQLite с 6 таблицами (users, groups, notification_settings, hours_history, schedule_changes, favorites)
- **Планировщик задач** - APScheduler для уведомлений и cron-задач
- **Кэширование расписания** - автоматическое обновление и сравнение изменений
- **Custom Bot API** - поддержка локального Bot API сервера для больших файлов
- **Логирование** - детальное логирование всех операций с цветным выводом
- **Обработка ошибок** - graceful shutdown, обработка сигналов SIGINT/SIGTERM
- **Тестирование** - 18 тестовых модулей с покрытием, CI/CD через GitHub Actions

## 🚀 Установка

### Предварительные требования

- Python 3.10, 3.11 или 3.12
- Git
- Токен Telegram бота (получите у [@BotFather](https://t.me/BotFather))

### Пошаговая установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/karst3nz/new_mtc_helper_bot
   cd new_mtc_helper_bot
   ```

2. **Создайте виртуальное окружение:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Linux/Mac
   # или
   .venv\Scripts\activate     # Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r req.txt
   ```

4. **Настройте конфигурацию:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env своими данными
   ```

5. **Запустите бота:**
   ```bash
   python main.py
   ```

### Получение Telegram ID

Чтобы узнать ваш Telegram ID:
1. Напишите боту [@userinfobot](https://t.me/userinfobot)
2. Скопируйте значение "ID" и используйте его как ADMIN_ID

## ⚙️ Конфигурация

Бот поддерживает два способа конфигурации: через переменные окружения (.env файл) или через config.ini.

### Способ 1: Переменные окружения (рекомендуется)

1. **Создайте файл `.env` на основе `.env.example`:**
   ```bash
   cp .env.example .env
   ```

2. **Отредактируйте `.env` файл:**
   ```env
   # Токен вашего Telegram бота (получите у @BotFather)
   BOT_TOKEN=your_bot_token_here
   
   # ID администратора бота (ваш Telegram ID)
   ADMIN_ID=your_admin_id_here
   
   # Настройка отправки расписания (1/0)
   SEND_RASP=1
   
   # ID чата для бэкапов (опционально, по умолчанию = ADMIN_ID)
   BACKUP_CHAT_ID=your_backup_chat_id_here
   
   # Telegram API credentials (опционально, для custom bot API)
   API_ID=your_api_id_here
   API_HASH=your_api_hash_here
   
   # URL кастомного Bot API сервера (опционально)
   CUSTOM_API_URL=
   
   # Режим отладки (True/False)
   DEBUG=False
   ```

### Способ 2: config.ini (для обратной совместимости)

Создайте файл `config.ini` на основе `config.ini.example`:

```ini
[config]
# Токен вашего Telegram бота (получите у @BotFather)
bot_token = YOUR_BOT_TOKEN_HERE

# ID администратора бота (ваш Telegram ID)
admin_id = YOUR_ADMIN_ID_HERE

# Настройка отправки расписания (1/0)
send_rasp = 1

# ID чата для бэкапов (опционально)
backup_chat_id = YOUR_BACKUP_CHAT_ID_HERE

# Telegram API credentials (опционально)
api_id = YOUR_API_ID_HERE
api_hash = YOUR_API_HASH_HERE

# URL кастомного Bot API сервера (опционально)
custom_api_url = 

# Режим отладки
DEBUG = False
```

**Примечание:** Переменные окружения имеют приоритет над config.ini. Если параметр указан в обоих местах, будет использовано значение из .env файла.

### Параметры конфигурации

| Параметр | Описание | Обязательный |
|----------|----------|--------------|
| `BOT_TOKEN` | Токен Telegram бота от @BotFather | Да |
| `ADMIN_ID` | Telegram ID администратора | Да |
| `SEND_RASP` | Отправка расписания (1/0) | Нет (по умолчанию 1) |
| `BACKUP_CHAT_ID` | ID чата для бэкапов БД | Нет (по умолчанию = ADMIN_ID) |
| `API_ID` | Telegram API ID для custom bot API | Нет |
| `API_HASH` | Telegram API Hash для custom bot API | Нет |
| `CUSTOM_API_URL` | URL кастомного Bot API сервера | Нет |
| `DEBUG` | Режим отладки (True/False) | Нет (по умолчанию False) |

## 🎯 Использование

### Для пользователей

1. **Начало работы:**
   - Отправьте `/start` боту
   - Выберите свою группу
   - Настройте дополнительные параметры

2. **Просмотр расписания:**
   - Используйте команду `/rasp` или кнопки меню
   - Быстрые команды: `/сегодня`, `/завтра`, `/неделя`
   - Inline-режим: `@bot расписание` в любом чате
   - Выберите дату через календарь
   - Получите расписание с эмодзи предметов

3. **Управление пропущенными часами:**
   - Команда `/hours` для учета пропусков
   - Просмотр графиков и статистики
   - Экспорт данных в Excel
   - Настройка уведомлений о пропусках

4. **Настройка уведомлений:**
   - Перейдите в Настройки → Уведомления
   - Включите ежедневное расписание
   - Настройте напоминания о парах
   - Установите порог для уведомлений о пропусках

### Для администраторов

1. **Административная панель:**
   - Команда `/admin` для доступа к панели
   - Управление пользователями и группами
   - Отправка объявлений
   - Просмотр статистики

2. **Мониторинг:**
   - Просмотр логов в файле `app.log`
   - Статистика использования бота
   - История изменений расписания

## 📱 Команды бота

### Основные команды

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/start` | Главное меню и регистрация | Все пользователи |
| `/settings` | Настройки (работает в группах) | Все пользователи |
| `/hours` | Управление пропущенными часами | Все пользователи |
| `/rasp [дата]` | Расписание на дату | Все пользователи |
| `/lesson_schedule` | Расписание звонков | Все пользователи |
| `/admin` | Административная панель | Только администратор |

### Быстрые команды

| Команда | Альтернатива | Описание |
|---------|--------------|----------|
| `/сегодня` | `/today` | Расписание на сегодня |
| `/завтра` | `/tomorrow` | Расписание на завтра |
| `/неделя` | `/week` | Расписание на всю неделю |

### Inline-режим

Используйте бота в любом чате без открытия диалога:

- `@bot расписание` - расписание на сегодня
- `@bot завтра` - расписание на завтра  
- `@bot 01.04` - расписание на конкретную дату (формат ДД.ММ)

## 🔧 Разработка

### Архитектура

**Паттерны:**
- FSM (Finite State Machine) для диалогов
- Декораторы для проверки прав и типов чатов
- Dependency Injection через config.py
- Repository pattern для работы с БД

**Асинхронность:**
- Все операции I/O асинхронные (aiohttp, aiosqlite, aiofiles)
- Graceful shutdown с обработкой SIGINT/SIGTERM
- APScheduler для фоновых задач

**Безопасность:**
- SQL injection защита через whitelist групп
- Валидация входных данных
- Ограничение flood control для рассылок

### Логирование

Логи сохраняются в `app.log` (кодировка windows-1251):

```bash
# Просмотр логов в реальном времени
tail -f app.log

# Поиск ошибок
grep "ERROR" app.log

# Последние 100 строк
tail -n 100 app.log
```

Пример использования в коде:

```python
from utils.log import create_logger

logger = create_logger(__name__)
logger.info("Информационное сообщение")
logger.warning("Предупреждение")
logger.error("Ошибка", exc_info=True)
```

### Cron-задачи

Автоматические задачи выполняются через APScheduler:

- **03:00** - Бэкап базы данных в Telegram
- **04:00** - Обновление данных пользователей
- **По расписанию** - Отправка уведомлений о парах

### Custom Bot API

Для загрузки файлов >50MB можно использовать локальный Bot API сервер:

1. Настройте `API_ID`, `API_HASH` в .env
2. Укажите `CUSTOM_API_URL` (например, `http://localhost:8081`)
3. Бот автоматически запустит сервер при старте


Расписание загружается с официального сайта: https://bseumtc.by/schedule/
